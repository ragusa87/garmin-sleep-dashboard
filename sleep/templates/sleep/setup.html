Connexion interactive recommandé

just login

L'e-mail et le mot de passe sont demandés au clavier (saisie masquée, MFA supporté) et ne sont jamais écrits sur disque. Seuls des jetons de session OAuth sont conservés, dans {{ tokenstore }} (permissions 0600, dans le dossier du projet). Ensuite :

just sync        # 30 derniers jours
just sync 90     # 90 derniers jours

Alternative : .env.local à éviter pour le mot de passe

Pour un usage non interactif (cron, conteneur), créer un fichier .env.local à la racine du projet (jamais commité) :

GARMIN_EMAIL=vous@example.com
GARMIN_PASSWORD=votre-mot-de-passe

Le mot de passe est alors en clair sur disque — préférer la connexion interactive puis monter le dossier de jetons. Redémarrer le serveur après création du fichier.

Pourquoi pas une clé API ?

Garmin ne propose pas de clé API personnelle : son API officielle (Health API) est réservée aux partenaires commerciaux approuvés. Cette application utilise donc le même flux de connexion que l'application mobile Garmin Connect, et n'en conserve que les jetons de session.

  • Activer la MFA sur votre compte Garmin — supportée au login.
  • Pour révoquer les jetons : changer le mot de passe du compte Garmin, ou supprimer le dossier {{ tokenstore }}.

Sans compte : export GDPR

Demander un export sur garmin.com, télécharger le zip reçu par e-mail, puis :

just import-zip /chemin/export.zip
